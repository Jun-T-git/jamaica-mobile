# PHILOSOPHY — 製品哲学とコア

このアプリの「なぜ」と「本質」を定義する最上位ドキュメント。**新機能・変更は、この不変条件を壊してはならない。** 迷ったらここに立ち返る。

## 1. プロダクトの本質

**Modern Jamaica（ジャマイカの木）** は、ボードゲーム「ジャマイカ」に着想を得た**数字パズルゲーム**。

- プレイヤーは **5 つの数字**と四則演算（＋ − × ÷）を使い、**式の木（expression tree）を組み立てて目標値に到達**させる。
- 5 つの数字を**すべて 1 回ずつ**使い切り、最後に残る**単一のノード（木の根）が目標値と一致**すれば正解。
- 生成される問題は**必ず解ける**ことが保証される（生成過程そのものが 1 つの解になっている）。

これがゲームの核。ここが変わると別のゲームになる。

## 2. 絶対に壊してはいけない不変条件（Invariants）

コア／ゲームロジックに触れる変更は、以下を**すべて**満たし続けること。破る場合は必ず [decisions/](./decisions/) に新しい ADR を書き、意図を残す。

1. **解の保証**: 生成されるすべての問題は、5 つの数字全部を使って目標値に到達できる。（`utils/problemGenerator.ts` の `calculateTarget` が生成過程＝解を保証）
2. **5 数字・全消費**: 問題は必ず 5 つの数字。すべて木の葉として消費され、最終的に木の根が 1 つ残る。
3. **正解判定の一貫性**: 正解は「残ったノードが 1 つ」かつ「その値と目標値の差が `< 0.001`」。浮動小数点誤差を吸収するためのこの閾値を勝手に変えない。（`store/gameStore.ts connectNodes`）
4. **非負・実用値**: 減算は絶対値、除算は割り切れる時のみ（`applyOperator`）。プレイヤーに提示される数は扱いやすい値に保つ。
5. **日本語ファースト**: UI テキストは日本語が正。将来の多言語化を見据えても、既定言語は日本語。
6. **オフラインでも遊べる**: コアのパズル体験（問題生成・木の構築・判定）はネットワーク非依存。ランキングと広告のみが外部依存。

## 3. インタラクションの哲学

- **タップで繋ぐ（tap-to-connect）**。ドラッグ&ドロップではない。「ノード → 演算子 → ノード」の 3 タップで結合。指の可動域が小さく、片手で完結し、誤操作が少ない。判断の経緯は [decisions/0001-tap-to-connect.md](./decisions/0001-tap-to-connect.md)。
- **やり直せる安心感**: Undo で一手戻せる。試行錯誤を歓迎する。
- **即時フィードバック**: 結合・正解・カウントダウンなどに効果音とアニメーション（RN コア `Animated`）を付け、操作の手応えを返す。

## 4. デザイン哲学

- **温かみのあるモダンダークテーマ**。Tetris Effect / Monument Valley / Alto's Odyssey に着想（`design/modernDesignSystem.ts` 冒頭コメント）。冷たい純黒ではなく、青みがかった暖かい暗色。
- **ファミリーフレンドリー & アクセシブル**: 高コントラストだが刺激的すぎない配色。Dynamic Type 対応（`getDynamicFontSize`）で文字拡大に耐える。
- **ミニマル**: 画面は `headerShown: false`。ゲーム世界に没入させ、装飾より余白と階層で語る。
- **モードごとの色人格**: チャレンジ＝暖色（coral/gold）、無限＝寒色（neon/mint）。難易度も色で区別（かんたん=mint / ふつう=neon / むずかしい=coral）。
- 具体トークンは [DESIGN-SYSTEM.md](./DESIGN-SYSTEM.md) を参照（数値をここで二重化しない）。

## 5. ゲームモードの意図

| モード | 体験の狙い | 特徴 |
|---|---|---|
| **チャレンジ** | 短時間の集中・スコア競争 | 制限時間・正解で時間ボーナス・スキップ 2 回・スコア計算・**ランキング対象** |
| **無限** | リラックスして数をこなす | 5 分固定・スキップ無制限・「正解数」を追う・ランキング非対象 |

難易度（かんたん/ふつう/むずかしい）は数字レンジと時間で難度を調整する軸。詳細は [GAME-CORE.md](./GAME-CORE.md)。

## 6. 収益とプライバシーの方針

- **収益は広告のみ**（AdMob）。コア体験を邪魔しない頻度（数ゲームに 1 回のインタースティシャル）。
- **個人情報を収集しない**。ユーザーは端末ローカル生成の匿名 ID（`user_<ts>_<rand>`）。公開されるのはニックネームとスコアのみ。（ルート `docs/`（プライバシーポリシー）と整合させること）

## 7. 品質のスタンス

- **リリース済みプロダクト**として、後方互換とデータ整合（ハイスコア・ランキング）を尊重する。
- ドキュメントより**コードが正**（[README.md](./README.md) 参照）。
