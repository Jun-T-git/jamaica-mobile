# Modern Jamaica エンジニアリングドキュメント

このディレクトリは、React Native アプリ **Modern Jamaica（ジャマイカの木）** の開発者向け常緑ドキュメント群です。AI（Claude Code）と人間の双方が、プロジェクトの哲学とコアを順守したまま自律的に開発するための土台です。

> **リポジトリルートの `docs/`** は別物（GitHub Pages のプライバシーポリシーサイト）です。混同しないこと。

## 第一原則: コードが正（Source of Truth）

**このアプリは既にリリース済み。実装が真実である。**
ドキュメントと実装が矛盾したら、**コードを正とし、ドキュメントを実装に合わせて直す**（逆はしない）。ドキュメントは実装を説明・要約・案内するものであって、実装を規定するものではない。

## ドキュメントの安定性ティア（freshness contract）

各ドキュメントは冒頭に `<!-- STABILITY: xxx -->` を宣言する。

| ティア | 意味 | 更新方針 |
|---|---|---|
| **canonical** | 常に実装と一致すべき「常緑」文書。高レベル・不変条件中心に**薄く**保つ。 | 実装を変えたら必ず追随。drift 検出の対象。 |
| **frozen** | ある時点の設計判断・提案の記録（ADR 等）。日付付き。 | **更新しない**。判断が変わったら新しい文書を追加する。 |
| **working** | 作業メモ・調査ノート・タスク計画。 | いつ消してもよい。長期保証なし。 |

**canonical を薄く保つ規律**: 正確な関数シグネチャや具体的な数値を散文で二重化しない。揮発性の詳細はコードに委ね、`file:関数名` の形でコードを指す。これがドキュメントが腐る最大の原因への対策。

## ドキュメントを腐らせない仕組み

- **更新規約**: 「どのコードを変えたらどの canonical 文書を更新するか」の対応表を [CLAUDE.md](../CLAUDE.md) と各文書に記載。
- **drift 検出**: `.claude/scripts/check-doc-drift.mjs` が「参照パスの実在」と「src が文書より新しくないか（git 更新時刻）」を機械チェック。
- **/sync-docs スキル**: 機能変更の完了時に呼ぶと、差分から影響 canonical 文書を逆引きし、`doc-keeper` サブエージェントが更新する。
- **Stop フック**: 作業終了時に drift 疑いを非ブロッキングで通知（nudge）。

詳細は [../.claude/](../../.claude/) と [CLAUDE.md](../CLAUDE.md) を参照。
